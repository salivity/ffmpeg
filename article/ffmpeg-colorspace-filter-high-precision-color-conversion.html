FFmpeg Colorspace Filter: High-Precision Color Conversion

Converting video between different color standards—such as BT.601, BT.709, and BT.2020—requires high precision to avoid color degradation, banding, and clipping. This article provides a straightforward guide on how to use FFmpeg’s dedicated colorspace filter to perform highly accurate color space conversions, detailing the essential parameters for primaries, transfer characteristics, and matrix coefficients.

Why Use the colorspace Filter?

While FFmpeg’s scale filter can handle basic color matrix conversions, the specialized colorspace filter performs calculations with higher internal precision (up to 14-bit or 16-bit). It also supports advanced dithering to eliminate color banding when bit-depth reduction occurs (e.g., converting from 10-bit to 8-bit).

Essential Parameters

To configure the colorspace filter, you must define the target color properties:

  • all: A shortcut parameter to set primaries, trc, and space to the same standard (e.g., bt709, bt2020).
  • primaries: Sets the color primaries (e.g., bt709, bt2020, smpte170m).
  • trc: Sets the Transfer Characteristics or gamma curve (e.g., bt709, iec61966-2-1 for sRGB, linear).
  • space: Sets the color matrix coefficients (e.g., bt709, bt2020nc, smpte170m).
  • range: Defines the color range, either tv (limited/mpeg) or pc (full/jpeg).
  • dither: Applies dithering to reduce banding. Options include fsb (Floyd-Steinberg), none, or ordered.
  • format: Specifies the output pixel format (e.g., yuv420p, yuv420p10le).

Practical Examples

Example 1: Standard SD (BT.601) to HD (BT.709) Conversion

When upscaling Standard Definition video to High Definition, you must convert the color space from BT.601 to BT.709 to prevent shifted, inaccurate colors.

ffmpeg -i input_601.mp4 -vf colorspace=all=bt709:format=yuv420p output_709.mp4

Note: Using all=bt709 automatically configures the primaries, transfer curve, and matrix space to the BT.709 standard.

Example 2: Explicitly Defining Custom Color Properties

If you need to mix different standards—such as converting a video to use sRGB transfer characteristics (IEC 61966-2-1) while retaining BT.709 primaries:

ffmpeg -i input.mp4 -vf colorspace=primaries=bt709:trc=iec61966-2-1:space=bt709:format=yuv420p output_srgb.mp4

Example 3: Precision BT.2020 (10-bit) to BT.709 (8-bit) with Dithering

When downsampling color spaces and bit depths (e.g., converting SDR BT.2020 10-bit down to BT.709 8-bit), applying Floyd-Steinberg dithering prevents noticeable color banding in gradients like skies and shadows.

ffmpeg -i input_2020_10bit.mp4 -vf colorspace=all=bt709:dither=fsb:format=yuv420p output_709_8bit.mp4

Example 4: Converting Color Range (Limited to Full)

If you need to change the color range from limited (TV) to full (PC range) while converting to BT.709:

ffmpeg -i input.mp4 -vf colorspace=all=bt709:range=pc:format=yuv420p output_full_range.mp4
