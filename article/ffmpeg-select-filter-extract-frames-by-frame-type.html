FFmpeg Select Filter: Extract Frames by Frame Type

This guide explains how to use the FFmpeg select filter to extract specific frame types—such as I-frames (keyframes), P-frames, and B-frames—from a video file. You will learn the exact command-line syntax for each frame type, how to preserve the correct presentation timestamps, and how to export the selected frames as high-quality images.

Understanding the select Filter and pict_type

FFmpeg identifies frame types using the pict_type constant within its evaluation engine. The three primary frame types you can target are: * I (Intra-coded frames / Keyframes) * P (Predicted frames) * B (Bi-directional predicted frames)

To filter by these types, you use the select video filter (-vf) with the eq (equal) function to match the pict_type variable.

Extracting I-Frames (Keyframes)

I-frames contain complete image data and do not require information from other frames to decode. To extract only I-frames, use the following command:

ffmpeg -i input.mp4 -vf "select='eq(pict_type,I)'" -fps_mode vfr output_%04d.png

Command Breakdown:

  • -vf "select='eq(pict_type,I)'": Tells FFmpeg to select only video frames where the picture type equals “I”.
  • -fps_mode vfr: Sets the frame presentation mode to Variable Frame Rate (VFR). This is critical because it prevents FFmpeg from duplicating the extracted frames to match the source video’s original frame rate. (Note: If you are using an older version of FFmpeg, use -vsync vfr instead).
  • output_%04d.png: Generates sequentially numbered PNG images (e.g., output_0001.png, output_0002.png).

Extracting P-Frames

P-frames hold only the changes in the image from the previous frame. To extract all P-frames, run:

ffmpeg -i input.mp4 -vf "select='eq(pict_type,P)'" -fps_mode vfr output_%04d.png

Extracting B-Frames

B-frames refer to both preceding and succeeding frames to calculate image changes, offering the highest compression. To extract all B-frames, run:

ffmpeg -i input.mp4 -vf "select='eq(pict_type,B)'" -fps_mode vfr output_%04d.png

Extracting Multiple Frame Types

You can combine selection criteria using logical operators. For example, to extract both I-frames and P-frames while ignoring B-frames, use the addition (+) operator which acts as a logical “OR”:

ffmpeg -i input.mp4 -vf "select='eq(pict_type,I)+eq(pict_type,P)'" -fps_mode vfr output_%04d.png

Maintaining Original Timestamps

If you need the output files to reflect their actual timestamps from the original video timeline rather than a simple sequential order (1, 2, 3…), you can write the presentation timestamps (PTS) directly into the filenames:

ffmpeg -i input.mp4 -vf "select='eq(pict_type,I)'" -fps_mode vfr -frame_pts true output_%d.png

Using -frame_pts true will name each output image according to its packet index number in the stream, allowing you to easily identify exactly when each frame appeared in the source video.
