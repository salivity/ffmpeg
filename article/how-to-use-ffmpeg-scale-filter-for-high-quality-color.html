How to Use FFmpeg Scale Filter for High Quality Color

When resizing video in FFmpeg, default settings can sometimes introduce color degradation, banding, or loss of detail. This article provides a straightforward guide on how to configure the FFmpeg scale filter with high-quality color conversion options, focusing on scaling algorithms, color matrix configurations, and dithering techniques to ensure your output looks visually pristine.

1. The Core High-Quality Scaling Command

To scale a video while preserving maximum color and detail, you must override FFmpeg’s default bilinear scaling algorithm and enable advanced software scaler (sws) flags.

The standard high-quality scaling filter string looks like this:

-vf "scale=1920:1080:flags=lanczos+accurate_rnd+full_chroma_inp+full_chroma_int"

Here is what these options do: * lanczos: This algorithm is highly recommended for downscaling and upscaling because it preserves sharpness and detail far better than standard bilinear or bicubic filters. * accurate_rnd: Forces the scaler to use accurate rounding calculations, reducing truncation errors in color values. * full_chroma_inp: Forces the input to be processed with full chroma resolution, preventing color loss at the beginning of the pipeline. * full_chroma_int: Keeps chroma processing at full resolution internally throughout the scaling process.

2. Setting the Correct Color Matrix and Range

When changing resolutions (for example, from 4K to 1080p), you must ensure the color matrix converts correctly. Standard Definition (SD) uses bt601, High Definition (HD) uses bt709, and Ultra High Definition (UHD/HDR) uses bt2020.

You can define these target color spaces directly inside the scale filter using out_color_matrix and out_range:

-vf "scale=1920:1080:flags=lanczos+accurate_rnd:out_color_matrix=bt709:out_range=tv"
  • out_color_matrix=bt709: Ensures the output colors are correctly mapped to the HD standard.
  • out_range=tv: Sets the output range to “broadcast limit” (16-235). Use pc (0-255) if you require full-range color (common for web-only content or PC gameplay).

3. Preventing Banding with Dithering

When downsampling color depth (such as converting 10-bit HDR video to standard 8-bit SDR), color banding can occur in gradients like skies or shadows. You can apply a dithering algorithm to smooth out these transitions.

Add the global -sws_dither flag to your command line:

-sws_dither a_dither

Alternatively, you can specify it within the filter chain if you are converting pixel formats using the format filter:

-vf "scale=1920:1080:flags=lanczos+accurate_rnd,format=pix_fmts=yuv420p" -sws_dither a_dither

4. Complete Command Example

To scale an input video to 1080p HD, convert the color space to BT.709, output to standard YUV 4:2:0 8-bit, and apply high-quality dithering to prevent banding, use the following command:

ffmpeg -i input.mp4 -vf "scale=1920:1080:flags=lanczos+accurate_rnd+full_chroma_inp+full_chroma_int:out_color_matrix=bt709:out_range=tv,format=yuv420p" -sws_dither a_dither -c:v libx264 -crf 18 -c:a copy output.mp4
